# 3yada AI Clinic (عيادة AI) — التقرير الشامل

**تاريخ التقرير:** 6 يونيو 2026  
**الإصدار:** 2.1.1  
**القاعدة:** SQLite (`better-sqlite3`)

---

## 1. ملخص تنفيذي

**عيادة AI** هي منصة طبية ذكية متكاملة (PWA + Android/iOS) تخدم ثلاثة أنواع من المستخدمين:
- **المريض**: مساعد أعراض ذكي → ملخص طبي → اقتراح أطباء → حجز موعد
- **الطبيب**: إدارة المواعيد، سجلات مرضى، وصفات طبية، حسابات، ملفات مرضى، اشتراكات
- **المشرف**: لوحة تحكم كاملة، إدارة المستخدمين والأطباء، استيراد/تصدير CSV، مراقبة المحادثات

## 2. هيكل التكنولوجيا (Tech Stack)

### 2.1 Frontend (الواجهة الأمامية)

| التقنية | الإصدار | الاستخدام |
|---------|---------|-----------|
| **React** | ^19.1.0 | مكتبة بناء واجهات المستخدم (UI Library) |
| **TypeScript** | ^5.8.3 | لغة برمجة مع أنواع (Type Safety) |
| **Vite** | ^6.3.5 | أداة بناء وتطوير (Build Tool & Dev Server) |
| **@vitejs/plugin-react** | ^4.4.1 | إضافة React لـ Vite |
| **Tailwind CSS** | ^4.1.8 | إطار عمل CSS utility-based للتنسيق السريع |
| **@tailwindcss/vite** | ^4.1.8 | إضافة Tailwind لـ Vite |

**ملاحظات:**
- **لا يوجد React Router** — التنقل يتم عبر state machine: `AppStep` في `App.tsx` (login → home → chat → summary → doctors → booking → confirmation + history, calendar, profile)
- **التوجيه عبر URL hash**: `#doctor`, `#admin`, `#patient`
- **لا يوجد State Management خارجي** — كل الحالة عبر `useState` و props drilling
- **التدويل (i18n)**: نظام مخصص بـ React Context، يدعم العربية (افتراضي) والإنجليزية مع تخزين الخيار في `localStorage`
- **الخط**: IBM Plex Sans Arabic (Google Fonts) للعربية، Inter/Segoe UI للإنجليزية

### 2.3 الذكاء الاصطناعي (AI/LLM)

النظام يدعم 3 مزودين للذكاء الاصطناعي، يختارهم عبر متغير `LLM_PROVIDER`:

#### أ. OpenRouter (افتراضي)
| الخاصية | القيمة |
|---------|--------|
| المكتبة | `fetch` مباشر (بدون SDK) |
| الموديل الافتراضي | `google/gemma-4-31b-it` |
| درجة الحرارة | 0.7 |
| أقصى حد | 2048 token |
| التنسيق | JSON Response Format |

#### ب. Google Gemini
| الخاصية | القيمة |
|---------|--------|
| المكتبة | `@google/generative-ai` ^0.24.1 |
| الموديل | `gemini-2.5-flash` |
| الميزة | `responseSchema` لإجابات JSON منظمة |

#### ج. Ollama (محلي)
| الخاصية | القيمة |
|---------|--------|
| الرابط | `http://localhost:11434` |
| الموديل الافتراضي | `qwen2.5` |
| الاستخدام | تشغيل محلي بدون إنترنت + استخراج نصوص الملفات |

**النظام البرمجي (System Prompt):** 62 سطر تعليمات بالعربية في `server/llm/prompt.ts`، يوجه المساعد الطبي لجمع الأعراض، توليد الملخص، وتحديث ملف المريض.

### 2.4 قاعدة البيانات (Database)

**15 جدولاً** في قاعدة SQLite واحدة:

| # | الجدول | الغرض |
|---|--------|-------|
| 1 | `users` | حسابات المرضى (id, name, email, password_hash, phone, age, gender, health_insurance, date_of_birth) |
| 2 | `sessions` | جلسات الاستشارة (id, user_id, title, summary_json, status) |
| 3 | `conversations` | محادثات داخل الجلسة (id, user_id, session_id, title) |
| 4 | `messages` | رسائل المحادثة (id, conversation_id, role, content, attachments) |
| 5 | `appointments` | مواعيد الحجز (id, user_id, doctor_id, session_id, slot, patient_name, phone, fee, status) |
| 6 | `doctors` | بيانات الأطباء (18 عمود: id, name, specialty, hospital, city, rating, fees, slots, email, phone, bio, documents, verification, insurances) |
| 7 | `subscription_plans` | خطط الاشتراك (3 خطط: مجاني 0 ر.س/10 مرضى، أساسي 99/شهري 50 مريض، مميز 199/شهري غير محدود) |
| 8 | `doctor_subscriptions` | اشتراكات الأطباء (id, doctor_id, plan_id, start/end_date, is_trial, auto_renew) |
| 9 | `admins` | المشرفون (id, name, email, password_hash, role) |
| 10 | `medical_summaries` | ملخصات طبية للمواعيد (id, appointment_id, doctor_id, user_id, content, edited_at) |
| 11 | `doctor_clinic_patients` | مرضى عيادة الطبيب (id, doctor_id, name, phone, age, gender, history, notes) |
| 12 | `prescriptions` | وصفات طبية (id, doctor_id, patient_id, medication_name, dosage, frequency, duration) |
| 13 | `clinic_visits` | زيارات المرضى (id, doctor_id, patient_id, visit_date, chief_complaint, diagnosis, treatment_plan) |
| 14 | `patient_files` | ملفات المرضى (id, doctor_id, patient_id, file_type, title, file_path) |
| 15 | `clinic_financial_transactions` | المعاملات المالية (id, doctor_id, type, category, title, amount, status, payment_method) |

**الميزات:** WAL mode, Foreign Keys ON, ترحيل تلقائي (migrations) للأعمدة الجديدة.
