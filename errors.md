* **Category:** Security
* **Location:** `android/app/src/main/assets/public/assets/index-CLITnfp0.js`
* **Description:** Storing sensitive information like JWTs or access tokens in LocalStorage makes them accessible via client-side JavaScript, increasing the risk of token theft via XSS.
* **Why It Matters:** LocalStorage is accessible via any client-side JavaScript file executing in the browser context. This includes scripts imported from external CDNs.
* **Risk:** If the application suffers from Cross-Site Scripting (XSS), attackers can execute scripts to extract authentication tokens from LocalStorage and hijack user accounts.
* **Remediation:** Store access tokens in-memory inside React context, and retrieve them using HTTP-Only, Secure, and SameSite cookies to protect them from scripts.
* **OWASP Mapping:** A01:2021-Broken Access Control
* **CWE Mapping:** CWE-922: Insecure Storage of Sensitive Information

#### Code Example
```
// Insecure:
localStorage.setItem("authToken", token);

// Secure:
// Cookied storage managed on the HTTP server response header:
// Set-Cookie: token=xyz; Secure; HttpOnly; SameSite=Strict
```

* **Description:** Detected the use of 'dangerouslySetInnerHTML' which directly bypasses React's virtual DOM sanitization, exposing the app to Cross-Site Scripting (XSS).
